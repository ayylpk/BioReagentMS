====================================================
  BioReagentMS  Git 操作指南
  写于 2026年7月6日
====================================================

以后每次更新项目，打开终端，按下面三步做就行。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  第一步：挑文件（git add）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

把改了的东西装进"包裹"里。

  命令：
    git add .

  意思：
    当前目录下所有变更，全装进去。
    .gitignore 已经帮你拦住了 .env、.venv 等不该提交的东西，放心用。

  如果想一个个挑着装：
    git add 具体文件名

  装完后可以看看包裹里有什么：
    git status


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  第二步：封箱（git commit）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

给包裹贴标签，写明这次改了什么。

  命令：
    git commit -m "写清楚你改了什么"

  例子：
    git commit -m "新增智能助手页面，修复application.yml敏感信息泄露"

  注意：
    -m 后面的引号里是提交说明，别偷懒写"更新"，以后翻记录看不懂的。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  第三步：发送（git push）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

把包裹推到 GitHub。

  命令：
    git push

  如果是第一次推这个项目：
    git push -u origin main

  推送完去 https://github.com/ayylpk/BioReagentMS 刷新一下，看到改动上去了就是成功了。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  完整一次操作大概长这样：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  cd F:\code\project\BioReagentMS

  git add .

  git commit -m "新增智能助手Chat页面，新增agent后端，修复配置泄露"

  git push

  三行命令，30秒搞定。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  常用救命命令（出事了再看）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  看当前改了啥：
    git status

  看具体改了什么内容：
    git diff

  撤销某个文件的修改（回到上次提交的状态）：
    git restore 文件名

  撤销 git add（从包裹里拿出来，但不丢修改）：
    git restore --staged 文件名

  看提交历史：
    git log --oneline

  标记某个文件以后都不要提交：
    打开项目根目录的 .gitignore，把文件名写进去


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  重要提醒
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ⚠ 每次 git add . 之前，想一下有没有新增的敏感文件。
  ⚠ application.yml 里的密码和密钥用 ${环境变量} 写法，别写死。
  ⚠ .env 文件绝对不能提交，.gitignore 已经替你拦了，别手贱去掉。
  ⚠ 不确定的时候，先用 git status 看一眼，别闭眼 git add . 然后 git push -f。
